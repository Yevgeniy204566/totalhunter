## Список провалов по порядку

### 1. Добавил is_water в RETURNING без стрика (27 апр)
**Баг:** Реки давали 1 кадр воды → ложный сдвиг → бот ходил вдоль берега  
**Нарушение:** ANTI-PATTERNS AP-32

### 2. is_beyond_beacon_line только canvas (27 апр)
**Баг:** Canvas дрейфовал от дробных шагов → линия маяка не срабатывала → бот уходил в воду  
**Нарушение:** ANTI-PATTERNS AP-33

### 3. pixels_per_step=20 хардкод (27-28 апр)
**Баг:** Реальный сдвиг миникарты = 12px, хардкод 20px → ошибка 67% → маяк в неверном месте  
**Нарушение:** ANTI-PATTERNS AP-34

### 4. Маяк не рисовался при глубоком нырке (28 апр)
**Баг:** if 0 <= px < w → при нырке 5+ шагов маяк уходил за пределы миникарты → не рисовался → fallback → вода

### 5. Backstop inland_steps+3 слишком мал (28 апр)
**Баг:** При нырках 20 экранов с прыжками backstop кончался до возврата → вода

### 6. Visual path не проверял берег (28-29 апр)
**Баг:** `_is_at_coast_now` вызывался только в fallback → в visual path нет проверки → бот проходил берег в воду

### 7. is_water в RETURNING (29 апр)
**Баг:** Реки = 1 кадр воды → снова ложный сдвиг → бот ходит вдоль берега

### 8. Визуальная вода в _is_at_coast_now (29 апр)
**Баг:** Visual water срабатывала не в точке маяка → сдвиг в неверной позиции → нарушение змейки

### 9. Canvas боковой дрейф от visual nav (29 апр)
**Баг:** _click_vec записывал боковой компонент → is_beyond_beacon_line срабатывала рано → 7 боковых шагов
