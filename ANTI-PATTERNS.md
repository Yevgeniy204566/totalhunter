# ANTI-PATTERNS.md — Запреты и Грабли

> Не тратить время повторно на эти решения.
> Обновляется командой **«Хангоф»**.

---

## 1. OCR времени марша — ЗАПРЕЩЕНО

**Что было:** Читать время марша Картера через pytesseract из скриншота.
**Почему провалилось:** Нестабильно, зависит от языка игры, Carter overlay перекрывает регион y=773. ACCEL_TIME_REGION (y=295) — другой регион, но тоже нестабилен.
**Решение:** Детерминированная формула `T_one_way = T_max / 2^N`. Игра сама защищает от лишних ускорений.

---

## 2. Template matching для кнопок диалога — ЗАПРЕЩЕНО в мультиязычном контексте

**Что было:** Искать кнопки «Исследовать»/«Открыть»/«Ускорить» по PNG-шаблону.
**Почему провалилось:** Шаблоны только русские → ломается на EN/CN/DE/etc.
**Решение:** HSV-детект цвета кнопки + геометрия региона. Клики по хардкод-координатам с `dialog_offset_y` для браузера.

---

## 3. Action Verification в критическом пути — УБРАНО

**Что было:** `_verify_action()` после каждого клика — ждать появления/исчезновения PNG-шаблона.
**Почему провалилось:** Шаблоны только RU, несовместимо с многоязычностью. Замедляет цикл.
**Решение:** Прямые клики по координатам, `return True`. Верификация только там, где реально нужна.

---

## 4. window_scaler.py — УДАЛЁН, не восстанавливать

**Что было:** Масштабировщик координат на основе одной якорной точки.
**Почему удалён:** Недостаточная точность, заменён двухточечной калибровкой.
**Решение:** Всегда `coord_manager` из `coord_manager.py`. REF_A=(90,925), REF_B=(1149,88).

---

## 5. CompassNavigator — устаревший, не использовать

**Что было:** «Рикошет» навигатор — нырок → сдвиг без центроидного анализа.
**Почему заменён:** Менее точный, не учитывает реальный угол берега.
**Решение:** `CoastalSnakeNavigator` — единственный актуальный навигатор.

---

## 6. tk.Tk() в calibration_ui — ЗАПРЕЩЕНО

**Что было:** Создание `tk.Tk()` внутри `calibration_ui.py`.
**Почему падает:** CTkApp уже является root → конфликт двух Tk инстансов.
**Решение:** `run_calibration(parent=self)` + `tk.IntVar(master=root, ...)`.

---

## 7. Принудительные дополнительные шаги в RETURNING — ЗАПРЕЩЕНО

**Что было:** Добавлять extra шаги к blind фазе возврата чтобы «дойти до берега».
**Почему ломает:** Нарушает инвариант ⊥ движения, бот уходит вбок от линии нырка.
**Решение:** Фиксить `coast_detect_radius` (рекомендовано ~25px вместо 50px).

---

## 8. OCR-верификация внутри _accelerate() — ЗАПРЕЩЕНО

**Что было:** 10 попыток × 0.5с поиска `speed up_2.png` внутри `_accelerate()`.
**Почему удалено:** Медленно, зависит от языка, избыточно.
**Решение:** Прямые N кликов по `ACCEL_USE_BTN`. Игра сама не даст нажать лишнее.

---

## 9. Счётчик сбросов как критерий конца списка склепов — ЗАПРЕЩЕНО

**Что было:** Считать количество `_reset_search()` и при N сбросах считать список пустым.
**Почему неверно:** Сброс может происходить по разным причинам, не только конец списка.
**Решение:** Детект конца по неподвижности меню (`image-diff`, `no_move_count >= 3`).

---

## 10. _max_march_sec как cap начального OCR-чтения — ЗАПРЕЩЕНО

**Что было:** Использовать `_max_march_sec` как верхний предел при OCR-чтении времени марша.
**Почему запрещено:** OCR удалён полностью. `_max_march_sec` — только для формулы ожидания и `calc_blind_leg`.
**Решение:** Детерминированная формула, без OCR.
