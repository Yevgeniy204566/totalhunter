## 1. OCR времени марша — ЗАПРЕЩЕНО

**Что было:** Читать время марша Картера через pytesseract из скриншота.
**Почему провалилось:** Нестабильно, зависит от языка игры, Carter overlay перекрывает регион y=773. ACCEL_TIME_REGION (y=295) — другой регион, но тоже нестабилен.
**Решение:** Детерминированная формула `T_one_way = T_max / 2^N`. Игра сама защищает от лишних ускорений.

## 2. Template matching для кнопок диалога — ЗАПРЕЩЕНО в мультиязычном контексте

**Что было:** Искать кнопки «Исследовать»/«Открыть»/«Ускорить» по PNG-шаблону.
**Почему провалилось:** Шаблоны только русские → ломается на EN/CN/DE/etc.

## 9. Счётчик сбросов как критерий конца списка склепов — ЗАПРЕЩЕНО

## 11. CORS `allow_origins=["*"]` с Authorization header — НЕ РАБОТАЕТ

**Что было:** `allow_origins=["*"]` в FastAPI CORSMiddleware.
**Почему не работает:** Браузер блокирует `Authorization: Bearer` вместе с wildcard origin — требует explicit origin.
**Решение:** Список конкретных доменов + `allow_credentials=True`.

## 12. HTTP API + HTTPS фронтенд = Mixed Content — БРАУЗЕР БЛОКИРУЕТ

**Что было:** Vercel (HTTPS) делает fetch к `http://34.68.86.57:8000`.
**Почему не работает:** Браузер блокирует HTTP-запросы со страницы HTTPS как "Mixed Content" — до CORS дело не доходит вообще.
**Решение:** Либо SSL на GCP (домен + nginx + Let's Encrypt), либо Vercel Rewrites как reverse proxy (запрос идёт server-side, минуя браузерную проверку).

## 14. Тест с misleading именем для ref_code happy path

**Что было:** `test_auth_google_with_valid_ref_code` — тест НЕ проверяет что `invited_by_id` реально устанавливается при валидном ref_code (тестирует только `ref_code=None`).

## 15. EMA angle clamp в CoastalSnakeNavigator — ЗАПРЕЩЕНО

**Что было:** `np.clip(diff_raw, -0.2, 0.2)` перед умножением на `coast_ema_alpha`.
**Почему ломает:** Замедляет адаптацию угла в 5×. Бот 25+ шагов идёт в неверном направлении (в воду) пока EMA догоняет реальный угол берега.
**Решение:** `self._coast_angle = (a + alpha * diff_raw) % (2 * np.pi)` — только нормализация, без clamp. `% (2π)` защищает от unbounded float accumulation.

## 16. Не сбрасывать `_steps_since_shift` при переходах состояний — ЗАПРЕЩЕНО

**Что было:** `_steps_since_shift` не сбрасывался при HOMING→DIVING и DIVING→RETURNING.
**Почему ломает:** Счётчик накапливается между нырками, `force_shift_after` срабатывает немедленно в начале нового нырка вместо накопленного лимита.
**Решение:** `self._steps_since_shift = 0` во всех 3 местах перехода.

## 17. `value_text` в темах GUI — отдельный ключ, не `primary`

**Что было:** Метки значений ползунков использовали `MD3["primary"]` для цвета текста.
**Почему плохо:** Deep Night primary = `#1B3A82` (тёмно-синий) — невидим на тёмном фоне `#141B2D`.
**Решение:** Отдельный ключ `value_text` в каждой теме: Dark Mode `#0D94D0`, Deep Night `#88AAFF`, Ocean `#38BDF8`, Light `#D4B483`. Аналогично `checkbox`/`checkbox_hover` — отдельные ключи.

## 18. SQLAlchemy async — обращение к ORM объекту после commit — ЗАПРЕЩЕНО

**Что было:** `order.id` читался снаружи блока `async with db.begin()`.
**Почему падает:** После выхода из `begin()` объекты expire. Доступ к атрибуту вызывает `MissingGreenlet` в async контексте.
**Решение:** Захватить нужные значения (`order_id_str = str(order.id)`) внутри транзакционного блока до его закрытия.

## 19. Free-Kassa: один секретный ключ — НЕВЕРНО

**Что было:** Использовать `FK_SECRET_WORD` и для построения ссылки оплаты, и для верификации вебхука.
**Почему неверно:** FK использует два разных ключа: `FK_SECRET_WORD` (первое секретное слово — для ссылки), `FK_SECRET_WORD2` (второе секретное слово — для HMAC проверки вебхука).

## 20. Referral % от base_amount вместо credits_total — ЗАПРЕЩЕНО

**Что было:** L1/L2/L3 реферальные % вычислялись от базовой суммы (без бонусных кредитов).
**Почему неверно:** Spec явно: реферальная база = `credits_total` (включая бонус). Например, pro = 2000cr (не 1500cr base).
**Решение:** `int(credits_total * rate)`, где `credits_total` = полная сумма пакета включая бонусы.

## 21. OCR-проверка масла с порогом 70K — УДАЛЕНА, не восстанавливать

**Что было:** `OIL_STOP_THRESHOLD = 70_000`, `parse_oil()`, `_read_oil()`, проверка `oil < OIL_STOP_THRESHOLD` перед отправкой капитана.
**Почему удалено:** OCR путает суффикс «М» (миллион) с «К» (тысяча) — 6.06М читается как 6.06К = 6060 < 70000 → ложный аварийный стоп. Ретраи не помогают: если шрифт/рендеринг стабильно даёт К вместо М, все N попыток дадут одно и то же. История чтений тоже не спасает: если глюк случился на первом походе — второго не будет.
**Решение:** Игра сама не даст отправить капитана при нулевом масле. `_send_captain` вернёт False → бот перезапустит цикл. Игра — единственный надёжный рубеж защиты.

## 22. Google Drive MCP — base64 через PowerShell — НЕ РАБОТАЕТ

**Что было:** `[Convert]::ToBase64String($bytes)` в PowerShell → передать в `create_file` MCP.
**Почему не работает:** PowerShell по умолчанию вставляет переносы строк каждые 76 символов → невалидный base64 → ошибка MCP.
**Решение:** Использовать `[Convert]::ToBase64String($bytes, [System.Base64FormattingOptions]::None)` или Python скрипт через Google Docs API напрямую.

## 23. Google Drive MCP `create_file` — не обновляет существующие Google Docs — ОГРАНИЧЕНИЕ

**Что было:** Попытка заполнить уже созданные Google Docs через MCP.
**Почему не работает:** MCP инструмент умеет только СОЗДАВАТЬ новые файлы. Метода update/patch нет.
**Решение:** Python скрипт с `google-api-python-client` → `documents().batchUpdate()` — единственный надёжный способ обновлять существующие Google Docs.
