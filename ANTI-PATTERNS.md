## ⛔ CANVAS / UI — Новые грабли (Хангоф #55)

**AP-CANVAS-1: Тёмный overlay поверх текстуры убивает всё — ЗАПРЕЩЕНО выше 0.55**
- Симптом: `rgba(0,0,0,0.82)` поверх реальной текстуры = чёрный прямоугольник, ни одна текстура не видна
- Причина: "эффект лака" звучит красиво, но > 0.55 alpha = полная непрозрачность визуально
- Правило: lacquer overlay MAX alpha = 0.50. Лучше меньше.

**AP-CANVAS-2: Тёмные цвета секторов (< #1428a0) = невидимые сектора**
- Симптом: колесо выглядит чёрным, цвета не различимы
- Причина: edge-stop градиента был `#1428a0` (почти чёрный синий) — на тёмном фоне не видно
- Правило: edge-stop сектора должен быть ярким: синий ≥ `#2C55EE`, зелёный ≥ `#008844`

**AP-CANVAS-3: Unsplash CORS — premium фото не грузятся без API key**
- Симптом: `img.onerror` → fallback, текстура не применяется
- Причина: `plus.unsplash.com` = платный тир, требует Unsplash API key
- Правило: использовать только `images.unsplash.com/photo-{ID}` (не plus), проверять что это свободное фото
- Лучшее решение для production: положить текстуры в `web/public/img/` и использовать локально

**AP-CANVAS-4: Итеративный рефактор UI без скриншота = трата времени**
- 6 версий колеса за одну сессию потому что не запрашивали скрин после каждой версии
- Правило: после любого визуального изменения — ПОПРОСИТЬ скрин. Не угадывать.

## ⛔ GIT / ДЕПЛОЙ — Новые грабли (Хангоф #54)

**AP-GIT-1: `__pycache__` в git-индексе — блокирует git stash/pull**
- Симптом: `git stash` / `git pull` падает с `error: Your local changes to __pycache__/xxx.pyc would be overwritten`
- Причина: `.pyc` файлы попали в индекс через `git add .` до создания `.gitignore`
- Решение: `git rm -r --cached __pycache__/ && echo "__pycache__/" >> .gitignore && git commit`
- **Теперь закрыто**: `.gitignore` создан в v1.3.0, все `.pyc` убраны из индекса навсегда

**AP-GIT-2: `git pull` на GCP с untracked migrations — падает merge**
- Симптом: `error: The following untracked working tree files would be overwritten by merge: server/alembic/versions/XXX.py`
- Причина: миграции создавались вручную через `psql` (без alembic) и не коммитились в репо, но потом появились в репо через другую сессию
- Решение: `sudo rm server/alembic/versions/XXX.py && sudo git pull`
- Файлы безопасно удалять — они придут из репо в том же виде

## 15. EMA angle clamp в CoastalSnakeNavigator — ЗАПРЕЩЕНО

**Что было:** `np.clip(diff_raw, -0.2, 0.2)` перед умножением на `coast_ema_alpha`.
**Почему ломает:** Замедляет адаптацию угла в 5×. Бот 25+ шагов идёт в неверном направлении (в воду) пока EMA догоняет реальный угол берега.
**Решение:** `self._coast_angle = (a + alpha * diff_raw) % (2 * np.pi)` — только нормализация, без clamp. `% (2π)` защищает от unbounded float accumulation.

## 20. Referral % от base_amount вместо credits_total — ЗАПРЕЩЕНО

**Что было:** L1/L2/L3 реферальные % вычислялись от базовой суммы (без бонусных кредитов).

## 21. OCR-проверка масла с порогом 70K — УДАЛЕНА, не восстанавливать

**Что было:** `OIL_STOP_THRESHOLD = 70_000`, `parse_oil()`, `_read_oil()`, проверка `oil < OIL_STOP_THRESHOLD` перед отправкой капитана.
**Почему удалено:** OCR путает суффикс «М» (миллион) с «К» (тысяча) — 6.06М читается как 6.06К = 6060 < 70000 → ложный аварийный стоп. Ретраи не помогают: если шрифт/рендеринг стабильно даёт К вместо М, все N попыток дадут одно и то же. История чтений тоже не спасает: если глюк случился на первом походе — второго не будет.
**Решение:** Игра сама не даст отправить капитана при нулевом масле. `_send_captain` вернёт False → бот перезапустит цикл. Игра — единственный надёжный рубеж защиты.

## 22. Google Drive MCP — base64 через PowerShell — НЕ РАБОТАЕТ

**Что было:** EMA (`coast_ema_alpha=0.3`, хардкод) + угловой демпфер (`max_pitch_delta=15°`) работали одновременно. Двойное ограничение давало ~1/3 реальной манёвренности: бот адаптировался к изгибу берега в 3× медленнее чем нужно.
**Почему запрещено:** Оба решают одну задачу (плавный поворот inland_vec), но суммируются в двойное торможение. Пользователь видит «ватное» управление.
**Решение:** Единый параметр `smooth_alpha` (0.1–1.0) управляет EMA напрямую. Демпфер (`_clamp_vec`, `_prev_inland_vec`) удалён. π-флип защита (diff > π/2) остаётся как жёсткий инвариант — не отключается.
**Рабочая конфигурация:** `smooth_alpha=0.70` — быстрая адаптация, проверено на 30+ локациях.

## 51. Блокировать `shift_vec` на всю сессию — ЗАПРЕЩЕНО (Хангоф #21)

**Что было:** `shift_vec` блокировался один раз при первом обнаружении берега. После 90° поворота `inland_vec` (изгиб берега) `shift_vec` оставался в старом направлении — становился анти-параллельным нырку. Сдвиг шёл в направлении нырка, а не перпендикулярно.
**Почему ломает:** Пользователь видел «нырок без сдвига» — сдвиг и возврат шли в одну сторону, визуально смещение отсутствовало.
**Решение:** `shift_vec = (-iv[1], iv[0])` пересчитывается при каждом переходе HOMING→DIVING по текущему `inland_vec`. `smooth_alpha` гарантирует плавность — осцилляции нет.

## 58. 🔒 Nuitka .pyd перекрывает .py — ИЗМЕНЕНИЯ В ИСХОДНИКЕ НЕ РАБОТАЮТ (2026-05-11)

**Решение:** `Remove-Item C:\BattleBot\crypt_hunter.cp313-win_amd64.pyd -Force` перед запуском из исходников.
**ПОСТОЯННОЕ РЕШЕНИЕ:** `build_release.py` теперь автоматически удаляет все `.pyd` из корня после сборки (шаг 7/7). После каждого `python build_release.py` корень чист.
**ПРАВИЛО:** `.pyd` в корне проекта = немедленно удалить. Они живут ТОЛЬКО в `dist/_internal/`.

## 60. Калибровка через AnyDesk — ВСЕГДА ДАЁТ СМЕЩЕНИЕ (Хангоф #52)

**Что было:** Пользователь подключился к машине клиента через AnyDesk и выставил первую калибровочную точку (Point A — джойстик). Бот кликал на пару сантиметров правее выставленной точки.
**Почему ломает:** AnyDesk отображает удалённый экран в масштабированном виде. Когда окно AnyDesk меньше реального разрешения экрана (например, 1920×1080 показывается в окне 1400px), курсор на экране клиента попадает не туда куда прицелился пользователь. PyAutoGUI читает реальные пиксели — они не совпадают с тем что видно в AnyDesk.
**Правило:** Калибровка ТОЛЬКО локально на машине клиента. Через AnyDesk — только наблюдение, не калибровка.
**Исключение:** Если AnyDesk → View → Scale → 100% (1:1 пиксель) — координаты точные. Но это редко возможно.

## 61. Перепаковка виджетов CTkTabview через pack(in_=...) — НЕ РАБОТАЕТ (Хангоф #53)
