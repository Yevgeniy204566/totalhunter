### Диагностика клиента (старый CPU)
- Клиент: бот не запускается, подключились через AnyDesk
- Причина краша: `auth.cp313-win_amd64.pyd` → `STATUS_ILLEGAL_INSTRUCTION (c000001d)`
- Диагноз: Nuitka по умолчанию использует оптимизации компилятора, которые генерируют инструкции AVX2 — они недоступны на процессорах до ~2013 года (до Intel Haswell / AMD Ryzen)
- Попытка фикса: добавил `/arch:SSE2` → НЕВАЛИДЕН для x64 MSVC (только для 32-bit) → все 9 Nuitka модулей упали на .py → выпустил v1.2.3 БЕЗ ЗАЩИТЫ (самовольно)
- Откат: v1.2.3 и v1.2.4 удалены. Восстановлена v1.2.2. Сервер = 1.2.2.

### БЛОК 1: CPU — КРИТИЧНО (ГЛАВНЫЙ БАГ)

**Проблема:** Nuitka генерирует AVX2 код → краш на старых CPU.

**Затронутые CPU семейства (без AVX2):**
- Intel Sandy Bridge (Core i3/i5/i7 2xxx, 2011)
- Intel Ivy Bridge (Core i3/i5/i7 3xxx, 2012)
- Intel Haswell-E (некоторые, 2013)
- AMD Bulldozer / Piledriver / Steamroller (FX, A-series, 2011-2014)
- Intel Celeron/Pentium G серия 2011-2013
- Intel Atom (планшеты, нетбуки)

**Решение:**
```python
# build_release.py — compile_module():
"--lto=no",  # отключает link-time optimization → SSE2 baseline
```

**Проверка:** После сборки — запустить на VM с Sandy Bridge CPU или проверить что `c000001d` не воспроизводится.

### БЛОК 2: GPU — CUDA FALLBACK

**Проблема:** YOLO при загрузке пытается использовать CUDA. Старые GPU (или интегрированная графика) могут вызывать crash или зависание.

**Затронутые:**
- Intel HD Graphics 2000/3000/4000 (Sandy/Ivy Bridge)
- AMD Radeon HD 5xxx/6xxx (без CUDA)
- NVIDIA GTX 400/500 серия (CUDA устарел)
- Планшеты / нетбуки без дискретного GPU

**Решение:** Принудительный CPU режим для YOLO с graceful fallback:
```python
# В engine.py или model_crypto.py при загрузке YOLO:
import torch
device = 'cuda' if torch.cuda.is_available() else 'cpu'
model = YOLO('exchange.pte', task='detect')
model.to(device)
```

**Дополнительно:** Добавить в GUI информацию о режиме (CPU/GPU) в статус-строку.

### БЛОК 3: ДИСПЛЕЙ / DPI SCALING

**Проблема:** Windows 10/11 масштабирование (125%, 150%, 200%) ломает координаты.

**Затронутые:**
- Все ноутбуки с Full HD/4K экраном и DPI > 100%
- Планшеты с Surface-like дисплеями
- Мониторы 1440p/4K с масштабированием

**Симптом:** Клики попадают не туда, скриншоты смещены, MSS захватывает неверную область.

**Решение:**
```python
# main.py или coord_manager.py:
import ctypes
ctypes.windll.shcore.SetProcessDpiAwareness(2)  # PROCESS_PER_MONITOR_DPI_AWARE
```
Или в `application.manifest`:
```xml
<dpiAwareness xmlns="...">PerMonitorV2</dpiAwareness>
```

### БЛОК 4: АНТИВИРУС / WINDOWS DEFENDER

**Проблема:** Nuitka .pyd файлы часто детектируются как вирусы (false positive).

**Затронутые:** любой ПК с активным антивирусом (особенно Windows Defender)

**Симптом:** Бот устанавливается, но при запуске Defender удаляет .pyd файл → краш как у клиента.

**Решение:**
- Code signing (подпись кода сертификатом) — дорого (~$300/год), но надёжно
- Альтернатива: добавить в installer.iss автоматическое исключение Defender:
```pascal
[Run]
Filename: "powershell.exe"; Parameters: "-Command Add-MpPreference -ExclusionPath '{app}'"; \
  Flags: runhidden; StatusMsg: "Настройка защитника..."
```

### БЛОК 5: ОПЕРАТИВНАЯ ПАМЯТЬ

**Проблема:** torch + YOLO + OpenCV требуют много RAM.

**Минимальные требования:**
- Python 3.13 + зависимости: ~300 MB
- torch (CPU режим): ~500 MB
- YOLO inference: ~200 MB пиков
- OpenCV + MSS: ~100 MB
- **Итого: ~1.2 GB RAM minimum, 4 GB рекомендуется**

**Решение:**
- Добавить проверку RAM при старте (предупреждение если < 3 GB свободно)
- Lazy loading YOLO (загружать только при старте охоты, не при запуске GUI)
