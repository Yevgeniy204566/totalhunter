## ЧТО НЕ РАБОТАЕТ

### 1. После скролла — клики промахиваются (КРИТИЧНО)

После scroll(-27 units), содержимое окна смещается на НЕПРЕДСКАЗУЕМОЕ количество пикселей.
Бот кликает по фиксированным координатам (GRID_Y + row_idx*SLOT_H + SLOT_H//2).
Но карточки после скролла НЕ находятся на этих позициях.
Результат: каждый клик после скролла идёт в пустоту.

ПОПЫТКИ РЕШИТЬ:
- scroll(-15): ~60px — слишком мало
- scroll(-27): ~264px — всё равно не точно
- scroll(-36): ~352px — не точно
- scroll(-45): ~440px — не точно
- Beacon offset: ищем маяк после скролла, вычисляем y_offset → в теории должно работать
- Результат beacon: не находит маяк (val=1 слишком распространённое)

ПРИЧИНА: pyautogui.scroll() на Windows нелинеен, не калиброван.

### 2. OCR пропускает карточки (КРИТИЧНО)

Слот [0,3] — count=4 — стабильно пропускается:
- Method A: НИКОГДА не читает
- Method B: читает '4' только на shift=-50, ОДИН раз из 5
- WEIGHTED_B требует 2 из 5 → не срабатывает
- Результат: бот никогда не кликает на эту карточку

### 3. Маяк ненадёжен

Маяк ставится на последнюю строку, первую карту.
Если val=1 (очень распространённое значение), поиск маяка после скролла
находит ПЕРВУЮ карту с val=1 — не обязательно тот же маяк.
