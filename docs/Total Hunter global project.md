# 🚀 Master Plan: Total Hunter (SaaS Infrastructure)
**Стадия:** Запуск коммерческой платформы.
**Объект работы:** Веб-интерфейсы и серверная логика GCP.

---

## 🏗 МОДУЛЬ 1: Pillar_Cloud_API (Серверный мозг на GCP)
*Бэкенд по адресу `34.68.86.57:8000` — это фундамент. Бот в него только стучится.*

1.  **Синхронизация Баланса:** Настройка эндпоинта `/use_credit`, который срабатывает в момент успешного действия бота. Сервер должен мгновенно обновлять таблицу в БД (PostgreSQL/SQLite).
2.  **Trial-логика:** Реализация эндпоинта `/claim_trial`. При первом входе нового HWID сервер выдает 300 кредитов и ставит метку «триал использован».
3.  **Логирование активности:** Сбор статистики от всех активных ботов (сколько кредитов списано за час/день) для вывода графиков в админку.

## 🔐 МОДУЛЬ 2: Pillar_Web_Platform (Сайт пользователя)
*Место, где пользователь превращается в клиента.*

1.  **Личный кабинет (Dashboard):**
    * **Авторизация через Google:** Использование OAuth 2.0 для входа на сайт.
    * **Статус:** Вывод текущего баланса и привязанного HWID.
    * **Управление HWID:** Кнопка «Сбросить привязку» (разрешено раз в неделю) — отправляет запрос на сервер для очистки поля HWID в профиле.
2.  **Магазин (Интеграция Free-Kassa):**
    * Выбор пакетов (например, 200, 1000, 4000 кредитов).
    * Генерация ссылки на оплату и обработка Webhook от кассы для мгновенного зачисления.

## 🌐 МОДУЛЬ 3: Pillar_Admin_Control (Твоя панель управления)
*Твой пульт управления бизнесом.*

1.  **Глобальный мониторинг:**
    * Счетчик онлайн-пользователей (кто стучался в `/heartbeat` за последние 5 минут).
    * Суммарный доход и количество найденных ресурсов по всей сети ботов.
2.  **Менеджмент:**
    * Поиск пользователя по Email или HWID.
    * Ручное добавление/списание кредитов.
    * Кнопка «Забанить» (блокировка доступа к API).
3.  **Broadcast System:** Окно для ввода текста. После нажатия «Отправить» сообщение улетает в БД и отображается у всех ботов в поле уведомлений при следующей проверке лицензии.

## 💰 МОДУЛЬ 4: Pillar_Economy (Реферальная система)
*Поскольку в боте уже есть вкладка «Рефералы», её нужно оживить на сервере.*

1.  **Генерация кодов:** При регистрации пользователь получает уникальный `ref_id`.
2.  **Бонусная логика:** Если новый HWID вводит чужой код при первом запуске, сервер начисляет бонус и пригласившему, и приглашенному.
