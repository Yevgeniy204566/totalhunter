## Идея 3 — Игра-трейдинг 🔥 САМАЯ ИНТЕРЕСНАЯ

### Суть
Мобильная игра (как Mario) где ландшафт = реальная цена крипто-актива.
- "Посадить цветок" = лимитный ордер
- "Прыжок" = рыночный ордер
- Стоп-лосс = игровой барьер
- Пользователь не видит скучных графиков — только игру

**Аудитория:** скальперы и быстрые трейдеры
**Блокчейн:** Monad (~1 сек финальность, EVM, дёшево)

### Архитектура (упрощённая — через API биржи)

```
Игра (React Native) → твой FastAPI сервер → API биржи на Monad → ордер
```

**Не нужно:** смарт-контракты, ликвидность, аудит — всё это делает биржа.
Берёшь уже существующий DEX/CEX на Monad с API и просто подключаешься.

### Монетизация
Не 0.5% от профита (сложно собирать через чужой API), а **подписка $5–10/мес** — проще и честнее.

### Стек
- React Native + Expo (знаем React)
- WebSocket цена: Binance API или DEX оракул бесплатно
- FastAPI backend — тот же стек что в Total Hunter
- Подписочная монетизация — та же логика

### Варианты по риску

| Вариант | Время | Что нужно |
|---|---|---|
| Paper trading (виртуальные деньги) | 4–6 нед | Только игра + FastAPI. Нет блокчейна. |
| Реальный трейдинг через API биржи | 6–8 нед | + интеграция с API DEX на Monad |

### Следующий шаг
1. Найти DEX на Monad с публичным API (Kuru, Ambient или другой)
2. Сделать прототип на 2 дня — показать 5 трейдерам
3. Если "да" — строить
