# Авто-калибровка — Design Spec
**Дата:** 2026-05-03  
**Статус:** Approved  

---

## Цель

Добавить кнопку **АВТОКАЛИБРОВАТЬ** на вкладку КАЛИБРОВКА рядом с существующей кнопкой КАЛИБРОВАТЬ.  
Бот автоматически находит обе якорные точки на экране и открывает лупу уже на найденных координатах — пользователь просто жмёт «Зафиксировать» без охоты за точками вручную.  
Ручная калибровка остаётся без изменений как запасной вариант.

---

## Архитектура

### Новый файл: `auto_calibration.py`

Единственная публичная функция:

```python
def auto_detect_points(
    screen_w: int,
    screen_h: int,
) -> tuple[tuple[int, int] | None, tuple[int, int] | None]:
    """
    Возвращает (point_a, point_b). Любая может быть None если не найдена.
    """
```

Внутри — два независимых детектора, вызываемых последовательно.

---

## Детектор Point A — белый прямоугольник джойстика

**Что ищем:** белый прямоугольный контур рамки навигационного джойстика миникарты (bottom-left).

**Алгоритм:**
1. Масштабируем REF_A=(90,925) на реальное разрешение:
   ```
   cx = int(REF_A[0] * screen_w / 1920)
   cy = int(REF_A[1] * screen_h / 1080)
   ```
2. Захватываем регион ±150px вокруг (cx, cy) через mss.
3. Конвертируем в grayscale → threshold (>200) → findContours.
4. Фильтруем контуры: ищем прямоугольный (approxPolyDP → 4 точки), area > 500px².
5. Центр найденного прямоугольника = point_a.
6. **Фоллбэк:** если контур не найден → возвращаем (cx, cy) (масштабированный REF_A).

---

## Детектор Point B — жёлто-зелёный плюс серебра

**Что ищем:** блочный жёлто-зелёный "+" который появляется в игре при наведении курсора на иконку серебра в ресурсной панели.

**Алгоритм:**
1. Масштабируем REF_B=(1149,88) на реальное разрешение:
   ```
   cx = int(REF_B[0] * screen_w / 1920)
   cy = int(REF_B[1] * screen_h / 1080)
   ```
2. Захватываем регион ±150px вокруг (cx, cy) — **baseline скриншот** (до наведения).
3. Перемещаем курсор в (cx, cy) через pyautogui.moveTo(cx, cy, duration=0.15).
4. Ждём 400мс.
5. Захватываем тот же регион — **hover скриншот**.
6. Diff: `new_pixels = hover - baseline` (абс. разница > 30 по яркости).
7. Если новых пикселей > 100: берём bounding box → его центр = point_b.
8. **Фоллбэк:** если diff пустой или < 100px → возвращаем (cx, cy).

## GUI — `main.py`

- Добавляем кнопку **АВТОКАЛИБРОВАТЬ** (синяя, height=40) **над** существующей кнопкой КАЛИБРОВАТЬ.
- Handler `_auto_calibrate()`:
  1. Скрываем главное окно (`self.withdraw()`).
  2. Определяем реальное разрешение экрана через mss.
  3. Вызываем `auto_detect_points(screen_w, screen_h)`.
  4. Возвращаем окно (`self.deiconify()`).
  5. Для каждой найденной точки — вызываем `_calibrate_one_point(root, found_pos, label)` из `calibration_ui.py`.
  6. Если обе точки найдены → `coord_manager.calibrate(a, b)` → обновляем статус.

## Изменения в `calibration_ui.py`

Минимальные. Добавляем два опциональных параметра в `run_calibration()`:

```python
def run_calibration(parent=None, start_a=None, start_b=None):
    # start_a/start_b: если переданы — лупа открывается на этих координатах
    # если None — поведение как сейчас (REF_A / REF_B)
    point_a = _calibrate_one_point(root, start_a or REF_A, "Точка А...")
    point_b = _calibrate_one_point(root, start_b or REF_B, "Точка Б...")
```

Существующий вызов `run_calibration(parent=self)` — без изменений в поведении.

## Обработка ошибок

| Ситуация | Поведение |
|----------|-----------|
| Точка A не найдена | Лупа открывается на масштабированном REF_A |
| Точка B не найдена (silver не hover-ится) | Лупа открывается на масштабированном REF_B |
| mss захват упал (нет экрана) | Показываем messagebox с предложением использовать ручную калибровку |
| Игра не открыта (baseline и hover одинаковые) | Попадает в фоллбэк Point B |
