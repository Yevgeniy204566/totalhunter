Архитектура Web Platform — 3 варианта

Выбери подход для фронтенда на Vercel

A

Чистый HTML + JS (без фреймворка)

Один HTML-файл, Google Identity Services, fetch к API. Деплой — просто закинуть файл на Vercel.

Плюсы

  • Нет сборки, нет npm
  • Максимально просто запустить
  • Легко читать и менять

Минусы

  • Быстро превращается в «спагетти»
  • Трудно масштабировать
B

⭐ React + Vite (рекомендую)

Современный SPA. Google OAuth через библиотеку. Vercel деплоит автоматически из GitHub.

Плюсы

  • Компоненты — легко добавлять экраны
  • Отличная экосистема для SaaS
  • Vercel + GitHub = деплой за 30 сек

Минусы

  • Нужен Node.js локально
  • Немного сложнее стартовать
C

Next.js + NextAuth

SSR + встроенный Google OAuth через NextAuth. Серверные компоненты.

Плюсы

  • OAuth «из коробки» — 10 строк кода
  • SEO, быстрый первый экран

Минусы

  • Избыточно для личного кабинета
  • Нужен домен для продакшн OAuth
  • Сложнее дебажить
⚠️ Важно для всех вариантов:
Google OAuth требует HTTPS на callback URL. Vercel даёт HTTPS бесплатно (total-hunter.vercel.app), поэтому OAuth будет работать. Бэкенд на GCP (HTTP) выступает как API — туда уходит уже верифицированный токен от Google, а не редирект.
